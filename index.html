Project Overview

Problem Statement & Objectives

Real-World Challenges

Traditional CCTV systems only record footage without providing real-time alerts. Enterprises require immediate detection of objects, tracking, people counting, fall events, fire/smoke, and facial recognition with minimal latency for timely response.

  • Heterogeneous camera streams with varying image quality.
  • Large volumes of video feeds, impossible to monitor manually 24/7.
  • Need for a unified platform supporting multiple vision tasks (detection, tracking, counting, pose, face, fire).

Project Objectives

Develop a unified AI vision platform that is easy to deploy, highly scalable, and fast enough for real-time applications.

  • High accuracy across six tasks: detection, tracking, counting, pose/fall, fire/smoke, and face recognition.
  • Low latency and stable performance when processing multiple camera streams concurrently.
  • Flexible deployment: on-premise (GPU/CPU) or cloud; rapid API integration.

Methodology & Architecture

How System Was Built

Pipeline & Methodology

Unified computer vision pipeline handling multiple tasks on the same video stream:

  • Video ingestion from cameras/RTSP streams.
  • Preprocessing: resizing, lighting normalization, noise reduction, aspect ratio preservation.
  • Multi-task inference: detection, tracking, counting, pose/fall, fire/smoke, face recognition.
  • Containerized deployment (Docker) with TensorRT acceleration.

System Architecture

Modular, loosely coupled components for scalability and model interchangeability:

  • Ingestion Layer: RTSP/camera stream handling and frame normalization.
  • AI Core: Core inference modules for detection, tracking, counting, pose/fall, fire, and face.
  • API & Service Layer: REST/gRPC endpoints for results and alerts.

Optimization & Acceleration Techniques

Inference Optimization and Deployment Stack

Optimized inference and streaming pipeline: ONNX, TensorRT, Docker, Kafka, gRPC, Triton, RTSP/FFmpeg, CUDA.

Acceleration & Infrastructure Stack

Multi-layer optimization combining model format, GPU acceleration, containerization, streaming, and RPC.

ONNX: Model format standardization for cross-platform inference.
TensorRT: GPU-optimized engine compilation for reduced latency.
Triton Inference Server: Efficient model serving with dynamic batching and ensemble support.
CUDA: GPU parallel computing for real-time processing.
Docker: Containerization for consistent deployment across environments.
Kafka: Distributed streaming and event queuing for ingestion & alerts.
gRPC: High-performance RPC for inference results and alerts.
RTSP + FFmpeg: Camera stream ingestion and transcoding as needed.

Performance Evaluation

Quantitative Benchmarks & Analysis

Evaluation Methodology

Each vision module was benchmarked independently and in multi-task configuration on an NVIDIA RTX 2080 Ti (12 GB) with TensorRT FP16 engines. Metrics were collected using Per Analyzer, measuring end-to-end inference latency, and throughput (FPS).

Key Metrics Explained

  • Concurrency: Number of simultaneous requests sent to the Triton server. Higher values increase load on GPU/CPU, simulating real-world multi-stream scenarios.
  • Throughput (inferences/second): Number of successful inferences processed per second. Higher is better ‚Äî indicates overall system efficiency.
  • Batch Size (bs4, bs6, bs8, bs16): Number of frames/samples grouped into a single inference call. Larger batches improve throughput when GPU resources are available, but excessive batching can lead to queueing or OOM if resources are saturated.
  • Chart Interpretation: Each line on the graphs represents throughput for a specific batch size as concurrency increases. Optimal operating points show peak throughput before saturation.
